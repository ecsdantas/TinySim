Welcome to TinySim!

TinySim is a powerful simulation tool currently under development. While it may still have some bugs or incomplete features, it's ready for you to explore and test its capabilities!

To begin, click on "Library" and drag components onto the workspace. Connect the ports and run your simulation. Use the search box at the top of the Library to filter blocks by name or by keyword (e.g. "filter", "logic", "control").

All wires must connect to ports
Output ports can have more than one connection.
Connections directly between wires are not supported.

General Tips:
  • Delete items: Select the item and press [delete] or [del] on your keyboard.
  • Flip horizontally: Select the item and press [I] on your keyboard.
  • Configure block: Select the block and press [O] on your keyboard, or click the gear icon.
  • Zoom: Use the scroll wheel to zoom in or out.
  • Zoom to fit selection: Select one or more items and press [Space].
  • Move item: Click and hold the item, then drag it to the desired position.
  • Move several items: Click and drag to select multiple items, then drag any selected item to move all of them.
  • Copy/Paste: Select one or more items and press [Ctrl+C] / [Ctrl+V].
  • Duplicate: Select one or more items and press [Ctrl+D].
  • Undo/Redo: Press [Ctrl+Z] to undo and [Ctrl+Shift+Z] to redo.
  • Save project: Press [Ctrl+S] to save your current project.
  • Double-click on wire: Create an intermediate point that can be dragged to form more complex paths.

You can simulate up to 10,000 steps. All simulations and circuit data are processed entirely on your device. We do not store any circuit data on our servers.

Simulation Settings

Click the gear/settings icon to open the simulation panel:

  • Step time / Stop time: Defines the simulation's sample time and total duration. The ratio between them cannot exceed the 10,000-step limit.
  • Real Time: Runs the simulation synced to the wall clock instead of as fast as possible. Step/Stop time fields are hidden while enabled.
  • Console Log: Prints each step's values to the browser console for debugging.
  • Integration method: Selects the numerical method used to solve dynamic blocks (Integrator, First Order).
  • Force stateless: Forces every block to be re-solved on each access, even if already solved. Use only if you run into solving issues, as it impacts performance.

Importing and Exporting Data

  • The CSV Import block reads a CSV file and exposes one output port per column. The file must have a column named Time; download the block's sample CSV for the expected format.
  • The CSV Export block writes the values it receives, alongside the simulation time, to a downloadable CSV file.
  • The Lookup Table block maps an input value to an output using a configurable table of points, with nearest, linear, or stepwise interpolation.

Subsystems

Group part of your diagram into a reusable Subsystem block:

  • Click View Inside on a Subsystem block to open its internal diagram, where you can place Input and Output marker blocks to define its ports.
  • Use the breadcrumb navigation to move back up to the parent diagram or the top-level diagram.
  • After adding or rearranging markers inside, click Sync Ports on the Subsystem block to update its external ports to match.
  • Subsystems are not yet supported in C code export.

Frequency Scope

The Frequency Scope block performs a Bode (frequency response) analysis of the linear chain feeding its input. Connect it to your signal path and click Analyze to view the magnitude/phase plot and the resulting transfer function.

Export to C Code

TinySim allows you to compile your project into C code, enabling further customization and integration into other systems. Keep the following in mind:

Important Considerations:
  • Always use a Display block as the output port for the generated C code.
  • Always use a Constant block as the input port for the generated C code.
  • All settings and parameters are copied into the generated C code.
  • Enable Real Time in the simulation settings to sync the generated code's execution speed to the simulation's sample time, instead of running as fast as possible.
  • The generated C code is divided into several files:
    • main.c: The entry point of your program. Customize this file as needed.
    • model.c: Contains the main model structure representing your diagram.
    • model.h: The header file for model.c, containing shared variables and functions.
    • libs.c: Standard TinySim functions related to diagram blocks.
    • libs.h: The header file for libs.c, containing shared variables and functions.
  • To compile the code, use: gcc *.c -o sim
  • To execute:
    • on windows: sim.exe
    • on linux: ./sim
